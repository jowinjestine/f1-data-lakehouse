# Human-in-the-Loop AI Workflow

## Philosophy

This project demonstrates a Human-in-the-Loop (HITL) AI development model where AI agents generate code, infrastructure, tests, and documentation, while human engineers control architecture decisions, security, quality gates, and deployment approval.

## Role Division

### AI Agent Responsibilities
- Generate Python code, dbt models, Terraform HCL, tests, documentation
- Create schema contracts, crosswalk seeds, CI/CD workflows
- Run linting, formatting, and test suites
- Create GitHub branches, PRs, and release tags
- Update Jira task descriptions and transitions
- Generate knowledge graphs and architecture analysis

### Human Engineer Responsibilities
- Approve architecture and design decisions
- Review and merge pull requests
- Approve IAM and security configurations
- Authorize backfill execution
- Validate dashboard narratives and metric definitions
- Final release approval
- GCP credential and secret management

## Review Gates

| Phase | Gate | Who Approves |
|---|---|---|
| 0. HITL Setup | Execution model | Human |
| 1. Foundation | Architecture + IAM | Human |
| 2. Thin Slice | One race weekend end-to-end | Human |
| 3. Full Ingestion | Backfill dry run | Human |
| 4. Full dbt | Lineage + metric definitions | Human |
| 5. Dashboard | Narrative + sharing settings | Human |
| 6. CI/CD | Deployment pipeline | Human |
| 7. Polish | Final release | Human |

## PR-Based Workflow

All work flows through GitHub pull requests:

1. AI agent creates feature branch
2. AI agent implements changes and opens PR with summary
3. CI pipeline runs (lint, test, terraform validate, dbt parse)
4. Human reviews PR diff and CI results
5. Human approves or requests changes
6. AI agent addresses feedback if needed
7. Human merges when satisfied

## Quality Controls

- **CI must pass** before merge (all 3 jobs: lint-and-test, terraform-validate, dbt-compile)
- **Secrets scanning** in CI prevents credential leaks
- **Push-to-main blocked** by branch protection — must use PRs
- **No destructive actions** without human approval (terraform apply, backfills, IAM changes)
